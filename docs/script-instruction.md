# InkFlow 视频脚本编写指南

本指南面向使用 InkFlow 生成的手绘/简笔科普视频，总结高完播、高互动脚本的核心写法，并提供可直接参考的范本。

---

## 1. 核心原则

- **一视频一主题**：每个视频只讲清楚一个问题，不要贪多。
- **开头即决战**：前 3 秒决定 80% 的留存，必须抛钩子。
- **画面即信息**：每句台词都要能对应一个具体、可画的视觉意象。
- **情绪 > 知识**：让观众"感觉学到了"比"真的学完了"更重要。
- **结尾有回响**：要么给结论、要么给反转、要么给续集期待。

---

## 2. 时长与钩子

### 2.1 基本标准

- 目标时长：≥2 分钟，信息密度高。
- **黄金 3 秒**：第一句必须抛钩子，激发好奇心，吊胃口。

### 2.2 常用钩子模板

| 模板 | 句式示例 | 适合选题 |
| --- | --- | --- |
| **反常识冲突** | "如果动物真的知道自己存在，那人类对它们做的每一件事，都是一场持续了几千年的犯罪。" | 科普、哲学、伦理 |
| **反直觉提问** | "你以五十码的速度冲向一辆停着的火车，和一辆五十码的火车冲向站着的你，伤害相同吗？" | 物理、数学、冷知识 |
| **第二人称假设** | "假如你一觉醒来，变成一头霸王龙，你能统治非洲草原吗？" | 生物、历史、幻想科普 |
| **悖论式答案** | "为什么老一辈的人害怕被人笑话？因为他们最喜欢笑话别人。" | 心理、社会、人际关系 |
| **痛点共鸣** | "废掉一个人，不需要灾难。" | 成长、心理、生活方式 |
| **悬念预告** | "科学家沉默了，因为这只能解释为一件事……" | 科学发现、动物行为 |

### 2.3 范本一：一面镜子，测出动物有没有自我意识

来源：抖音 @万物识，约 4 分 40 秒。

**黄金 3 秒钩子：**

> 如果动物真的知道自己存在，那人类对它们做的每一件事，都是一场持续了几千年的犯罪

用强冲突 + 道德悬念开场，立刻制造"这不可能是真的吧？"的好奇心。

**为什么适合当范本：**

- **时长充实**：4 分 40 秒里没有水分，每段都推进一个新论点。
- **故事驱动**：每个抽象概念都落到具体动物案例上（镜子测试的黑猩猩、嗅觉测试的狗、元认知实验的猕猴、虎鲸塔勒阔的 17 天哀悼）。
- **节奏转折多**："然而这个测试可能根本就是错的""但知道自己是谁和知道自己活着是两回事""然而科学在这里撞上了一堵墙"，不断打破观众预期。
- **情绪弧线完整**：从好奇 → 震惊 → 反思 → 沉重，结尾用黑猩猩华秀打出"你哭"回到情感共鸣。

**可借鉴的写作模板：**

1. 开头抛出反常识结论或道德冲突（钩子）。
2. 用一个经典实验/观点引入主题。
3. 连续用 2–3 个"反例"或"转折"深化话题。
4. 用最新科学共识（如《剑桥意识宣言》）提升可信度。
5. 结尾回到个体故事，让观众带走情绪而不是知识点。

### 2.4 范本二：相同速度下，你撞火车和火车撞你，伤害一样吗

来源：抖音，约 1 分 53 秒。

**黄金 3 秒钩子：**

> 你以五十码的速度冲向一辆停着的火车，和一辆五十码的火车冲向站着的你，你觉得自己受到的伤害相同吗？

用**反直觉选择题**开场，立刻让观众站队，评论区天然有讨论欲。

**为什么适合当范本：**

- **强互动设计**：先提问 → 让观众二选一 → 再看实验验证，观众被"押注"后更愿意看完。
- **视觉奇观中间件**：用"火车撞牛，牛原地爆炸"的极端画面作为认知锚点，比直接讲公式更有记忆点。
- **原理反转干净利落**：从"火车动能大"的直觉，切换到伽利略相对性原理，指出参照系才是关键。
- **结尾玩梗留印象**：用"祖传线粒体觉醒截停火车"的荒诞假设收尾，让严肃物理变得好笑、好传播。

**可借鉴的写作模板：**

1. 用反直觉问题或选择题开场，让观众先猜答案。
2. 用一个夸张、具象的实验/画面验证直觉，制造冲击。
3. 抛出一个简洁的科学原理，解释为什么直觉是错的。
4. 用幽默或极端假设做结尾，降低认知负担，增加分享欲。

### 2.5 范本三：假如你一觉醒来，变成一头霸王龙

来源：抖音，约 10 分 33 秒。

**黄金 3 秒钩子：**

> 假如你一觉醒来变成一头霸王龙，你能统治非洲草原吗？

用**第二人称沉浸式假设**开场，让观众立刻代入主角视角。

**为什么适合当范本：**

- **强代入感**：全程用"你"叙事，从破壳、被胡狼追、被行军蚁蛰，到最终成长为草原之王，观众在"打怪升级"中自然追完。
- **成长弧线清晰**：体长从 1 米 → 1.5 米 → 2 米 → 2.5 米 → 4 米 → 7 米 → 12 米，每个阶段都有对应冲突和情感事件。
- **情感锚点扎实**：母蜜獾护崽牺牲、与小蜜獾短暂相伴、为它复仇等情节，让硬核生存故事有了温情和遗憾。
- **续集钩子自然**：结尾从霸王龙死亡切换到翼膜重生，直接预告下一期，提高关注率。

**可借鉴的写作模板：**

1. 用"假如你变成 X"建立第二人称假设情境。
2. 设计一个清晰的成长/升级阶梯，每个阶段配一个具体危机。
3. 在升级过程中插入 1–2 个情感锚点，避免变成纯爽文。
4. 主角达到一定高度后，安排一个"身份揭晓"或"巅峰对决"时刻。
5. 结尾留反转或续集钩子。

### 2.6 范本四：为什么老一辈的人总是害怕别人笑话

来源：抖音，约 3 分 16 秒。

**黄金 3 秒钩子：**

> 为什么老一辈的人害怕被人笑话？因为他们最喜欢笑话别人。

用**悖论式答案**开场，先给出一个听起来"扎心"甚至"大逆不道"的结论，立刻抓住注意力并激发反驳欲。

**为什么适合当范本：**

- **论点鲜明**：把"面子"定义为熟人社会的"硬通货"，全村实时更新的公开数据，让观众秒懂老一辈的行为逻辑。
- **比喻接地气**：用"游戏版本机制""服务器设定""豪猪的刺""河豚的膨胀"等比喻，把抽象的社会心理学讲得像游戏攻略。
- **例子生活化**：村里王大妈评论李家儿媳妇、女朋友问"我胖了吗"、大城市地铁上摔一跤，全是观众身边熟悉的场景。
- **结构层层递进**：从现象 → 机制（熟人社会） → 对比（陌生人社会） → 心理学原理（投射） → 自我解放，逻辑闭环。
- **结尾给出口**："当你停止笑话别人的时候，你也就自动解除了对被别人笑话的恐惧"，把社会观察变成个人行动指南。

**可借鉴的写作模板：**

1. 提出一个反直觉或扎心的核心结论作为钩子。
2. 用一个系统/机制解释（如版本机制、经济逻辑、社会结构）说明为什么存在这种现象。
3. 用 2–3 个生活化例子让观众产生"对对对，就是这样"的共鸣。
4. 引入一个心理学/社会学概念提升深度。
5. 结尾把话题拉回到观众自身，给出一个可实践的启发。

## 3. 字幕与台词

- 每句不要太长，一口气读完。
- 一屏一句，不要堆长段落。
- 口语化，像对朋友讲故事。
- **字幕结尾不要加句号**，保持干净、利落，符合短视频平台习惯。
- 避免学术长句，多用短句、设问、停顿。

---

## 4. 画面节奏

- 每个 scene 停留 2.5–4 秒。
- 每个 shot 总时长不要超过 12 秒（Seedance 单段上限）。
- 可以把 2–4 句相关台词合并到一个 shot，避免 PPT 式切换。
- 每句台词对应具体视觉意象。
- 高潮场景（打斗、对决、反转）可以适当放慢，用 1–2 个长 shot 强化张力。

---

## 5. 画幅选择

- **知识/叙事类**：横屏 `1920x1080`（16:9），适合 B 站、YouTube。
- **短平快/强情绪类**：竖屏 `1080x1920`（9:16），适合视频号、小红书、抖音。
- 同一选题建议做两版：横版发 B 站，竖版发短视频平台。

## 6. 脚本创作流程（分四步 + 用户审阅）

脚本不是一次性写成最终 `script.json`，而是分步完成，确保画面、运动与叙事上下文一致。

**第零步：输出纯文本脚本供用户审阅（新增）**

在写 `script.json` 之前，先交付一份只有台词文本的纯文本脚本：

- 文件路径：`projects/<name>/scripts/script_text.md`
- 内容只包含旁白/字幕文本，每句一行，不标注时间戳，不写 `start_frame_prompt` 或 `video_motion_prompt`。
- 目标时长 2 分钟以上，信息密度高，钩子前置。
- **必须等用户确认文本后，再继续下一步。**

> **为什么这样做**：画面和运动提示词生成成本高、修改麻烦。如果台词本身方向不对，越早返工成本越低。纯文本审阅是防止"生成完才发现故事不对"的关键防线。

**第一步：纯文本剧本**

用户确认 `script_text.md` 后，将其转换为 `script.json` 的 `scenes` 数组：

- 每一句台词对应一个 `scene`。
- 只填 `scene_id`、`subtitle`、`duration_hint`，**不写 `start_frame_prompt` 或 `video_motion_prompt`**。
- 暂时不决定 shot 划分。

**第二步：划分 shot**

根据画面连续性与节奏，把 scene 分组为 shot。原则：

- 同一空间、同一主体、同一情绪弧的连续台词放进一个 shot。
- 每个 shot 总时长按 `duration_hint` 估算，控制在 12 秒以内。
- 转场、换主体、换情绪时切新 shot。

**第三步：跑 TTS 拿到真实时间序列**

写完 shot 分组和台词后，先跑音频：

```bash
uv run python main.py projects/<name> --step audio
```

跑完后查看 `logs/script_with_duration.json`，得到每句台词的真实秒数。然后：

- 把同一 shot 下所有 scene 的 `actual_duration` 相加，得到真实 shot 时长。
- 如果某个 shot **超过 12 秒**，必须拆 shot 或精简台词。
- 把真实时长写回 `script.json` 的 `duration_hint`，方便下次直接读取。

> **必须做这一步**：`duration_hint` 只是估算，TTS 真实时长可能更长。如果不先验证，Seedance 生成 12 秒视频后，后面合成时只能靠循环/裁剪硬对齐，效果会变差。

**第四步：补全画面与运动提示词**

确认 shot 分组和时长都 OK 后，再补全 `start_frame_prompt` 和 `video_motion_prompt`：

| 策略 | 字段 | 说明 |
| --- | --- | --- |
| **复用视频片段** | `hold_video: true` | 当前 shot 与上一 shot 画面几乎不变，只换台词，复用同一段视频。 |
| **全新 shot** | `use_reference_image: false`, `hold_video: false` | 需要全新的视觉意象。 |
| **图生图参考** | `use_reference_image: true`, `reference_from: "prev"` | 当前 shot 与上一 shot 在视觉上有**关联性或连续性**，例如同一人物换表情、同一场景加物件、同一空间换机位。参考指令要明确写出变化，如"给这张图片增加一个穿白大褂的男人"、"让这个人露出开心的表情"。 |

判断 shot 策略时要考虑前后 2–3 句的上下文，避免同一人物或场景被反复切镜头。

## 7. 画面提示词语言与内容

### 7.1 语言

- 所有 `start_frame_prompt` 和 `video_motion_prompt` 默认**用中文撰写**，如果用户要求，则使用英文。

### 7.2 内容

- `start_frame_prompt` 只描述**首帧画面内容**：主体、动作、场景、构图、情绪。
- `video_motion_prompt` 只描述**运动**：镜头怎么动、主体怎么动、环境如何变化。
- **禁止**在提示词里写风格词，例如"涂鸦风格""手绘""线条画""油画""写实""卡通""白色背景""电影感镜头"等。
- 画风、笔触、色调、背景质感由 `metadata.style_prompt` 和项目根目录的 `visual-reference.png` 统一控制，避免单句提示词覆盖全局画风。
- 参考上一张图时，提示词只需说明"发生了什么变化"，同样不写风格。

### 7.3 系统级动画风格提示词 `video_system_prompt`

- `metadata.video_system_prompt` 用于统一全片动画系统，例如："传统手绘动画风格，拍二动画，低帧率，断奏动作，可见的铅笔纹理，粗线条艺术，逐帧美感"。
- 该字段会自动追加到每条 `start_frame_prompt`、`video_motion_prompt`、legacy `image_prompt` 和封面图 prompt 的**前面**，不需要在每个 shot/scene 里重复写。
- `start_frame_prompt` 和 `video_motion_prompt` 仍然只写内容/运动，不写风格词；系统级风格由 `video_system_prompt` 和 `style_prompt` 共同承担。

## 8. 画面复用与参考

### 8.1 全局画风参考图 `visual-reference.png`

- 每个项目根目录下必须放置 `visual-reference.png`。
- **所有首帧图默认都是图生图**：生成任何 shot 的首帧时，都会把 `visual-reference.png` 作为参考图传入 Seedream，确保全片画风统一。
- 如果项目目录下没有 `visual-reference.png`，会 fallback 为不带参考图生成，并打印 warning。

### 8.2 `hold_video` 与 `use_reference_image`

- `hold_video: true`：当前 shot 复用上一 shot 的视频片段，不再调用 Seedance。
- `use_reference_image: true` + `reference_from: "prev"`：当前 shot 的首帧需要与上一张图保持**视觉连续性**（如人物换表情、同场景加物件）。此时会用上一张图覆盖全局画风参考图，优先保证画面连续性。
- 不依赖上一张图的 shot 会并发生成，提升效率；依赖 `prev` 的 shot 按顺序串行生成。
- 并发数通过环境变量 `SEEDREAM_MAX_WORKERS` 配置，默认 `32`。Seedream 文档标注的限流为 **500 IPM（张 / 分钟）**，可根据网络情况和账号配额调整，但不要超过该上限。
- 画面规划完成后，再生成最终 `script.json`。

## 9. 选题自检清单

在动手写脚本前，先回答这 5 个问题：

1. **核心问题是什么？** 一句话能说清吗？
2. **前 3 秒钩子是什么？** 能立刻让观众想继续看吗？
3. **有没有一个"哇"画面？** 即使用静音看，也有冲击力。
4. **结尾带走什么？** 知识、情绪、还是期待下一集？
5. **每个论点是否有具体例子？** 避免空谈概念。

如果 5 个问题都能明确回答，这个选题就适合做成 InkFlow 视频。
